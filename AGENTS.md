# Event Trackings — 通用规范

本仓库用于自动化跟踪多个国际事件，定期生成结构化风险日报。

## 通用规则

### 输出语言
- 所有日报和分析报告：**简体中文**
- 技术文档和配置文件：英文

### 写作风格
- 面向决策，不写论文式长文
- 结论先行，证据跟随
- 每条判断必须附带来源引用
- 不确定的判断必须明确标注不确定性
- 使用具体日期，禁止使用"今天"、"最近"等含糊时间词（除非紧跟具体日期）

### 来源要求
- 优先使用官方来源和主流权威媒体
- 重大事实性声明至少需要两个独立来源确认
- 如果来源质量不足，必须在"不确定性"部分说明
- 详细来源优先级参见各事件 skill 中的 `references/source-priority.md`

### 时间窗口
- 默认覆盖最近 24 小时
- 如信息不足，可扩展至 72 小时，并在报告中说明

### 文件写入规范
- 日报写入路径：`events/<event-name>/reports/YYYY/YYYY-MM-DD.md`
- 同时更新：`events/<event-name>/reports/latest.md`
- 同时维护：`events/<event-name>/reports/index.json`
- 按年份分目录存储

### 诚实原则
- 如果运行环境无法联网，必须明确说明，不得假装掌握最新情况
- 不得编造军事部署数量、官方声明或任何事实
- 即使无重大变化，也必须照常生成当天日报并标记"持平"

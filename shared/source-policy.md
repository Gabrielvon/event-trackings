# 通用来源政策 (General Source Policy)

本政策适用于 `event-trackings` 下的所有事件跟踪模块。遇到具体事件时，请以该事件 skill 内的 `source-priority.md` 为准，本政策作为跨事件的通用底层要求。

## 通用质量门槛
1. **优先使用一手官方信息**：政府部门、军方、海事/航空管理机构的首发声明或通告。
2. **权威媒体验证**：如果不涉及直接的官方通报，而是地缘事件或交叉领域的复杂局势，至少引用一家信誉良好的国际主流媒体（如 Reuters, AP, Bloomberg, FT, WSJ）。
3. **隔离宣传噪声**：区分官方通报与媒体宣传。

## 交叉验证与不确定性
- 重大事实性声明（如发生了交火、签署了封锁令）需要**至少两个独立且权威的新闻源或首发信息**予以确认。
- 如果来源质量不足，但事件极具影响，必须在最终报告的"不确定性"部分明确标出信息待核验。

## 不同来源类型的权重
- **官方文书/发布会**：最高权重。但需要考虑该官方立场的偏差。
- **国际通讯社（Reuters/AP/AFP）**：高权重。用于确认事实脉络。
- **专业智库分析（ISW, CSIS等）**：中高权重。主要用于补充背景和印证实地趋势。
- **社交媒体流传视频/消息**：最低权重。除非有权威开源情报机构（OSINT）背书，否则不可作为核心支撑证据，仅可写为“据网络视频流传”。

## 与具体事件专属策略的联动
各事件 skill 需要独立准备 `references/source-priority.md`，并在其中写明具体涉及哪些官方机构和智库。
